## 1. 概述

### 1.1 项目背景

构建一套**动态知识管理**系统，支持用户自定义知识表结构、层级化组织知识数据，并基于知识数据构建**知识图谱**，实现知识的结构化存储、检索与可视化展示。

### 1.2 项目目标

1. 提供灵活的**动态表结构**定义能力，无需编码即可创建和管理知识表
2. 支持**层级化**的知识分类体系
3. 提供**地图联动**展示能力
4. 支持**全局模糊搜索**，快速定位知识数据
5. 构建**知识图谱**，支持手动定义节点与关系，并进行可视化展示

### 1.3 技术框架

| 技术 | 版本/说明 |
|------|-----------|
| JDK | 8 |
| Spring Boot | 2.2+ |
| Spring Boot JPA | 与 PostgreSQL 配套的 ORM |
| PostgreSQL | 10（不使用 PostGIS） |
| MinIO | 对象存储（图片/文件） |
| 前端 | Vue.js（推荐，可替换） |
| 图谱可视化 | ECharts / D3.js / G6（推荐 AntV G6） |

### 1.4 设计原则

- **动态性**：表结构、字段均可动态配置，无需重新部署
- **层级化**：分类与知识表均支持树形层级结构
- **可扩展**：字段类型可扩展，预留扩展接口
- **轻量级**：不引入额外中间件，仅基于指定的 PostgreSQL + MinIO

### 2.2 知识分类管理

#### 2.2.1 功能描述

支持按**树形层级**管理知识分类，每个分类下可挂载多个知识表。

#### 2.2.2 详细功能

| 功能点 | 说明 |
|--------|------|
| 新增分类 | 支持在任意层级下新增子分类 |
| 编辑分类 | 修改分类名称、排序等 |
| 删除分类 | 分类下无知识表时允许删除；有知识表时提示并确认 |
| 分类树展示 | 树形结构展示所有分类，支持展开/折叠 |
| 拖拽排序 | 支持拖拽调整分类层级和顺序（可选） |

#### 2.2.3 分类属性

| 属性名 | 类型 | 说明 |
|--------|------|------|
| 分类名称 | 文本 | 必填，同一父级下不可重名 |
| 父级分类 | 引用 | 为空则为根节点 |
| 排序号 | 数值 | 同级排序 |
| 分类图标 | 图片 | 可选 |
| 分类描述 | 文本 | 可选 |

### 2.3 知识表管理

#### 2.3.1 功能描述

用户可在任意分类下创建知识表，定义该表的字段结构，系统根据定义动态生成数据存储入口。

#### 2.3.2 知识表 CRUD

| 功能点 | 说明 |
|--------|------|
| 新增知识表 | 指定名称、所属分类，创建时自动生成对应的数据存储结构 |
| 查看知识表列表 | 按层级展示所有知识表，**每个表显示数据总量** |
| 编辑知识表 | 修改表名称、描述等信息 |
| 删除知识表 | 删除表定义及其下所有数据，需二次确认 |
| 表详情 | 查看表字段结构，预览最近数据 |

#### 2.3.3 知识表属性

| 属性名 | 类型 | 说明 |
|--------|------|------|
| 表名称 | 文本 | 必填 |
| 所属分类 | 引用 | 必填，关联分类树 |
| 表描述 | 文本 | 可选 |
| 排序号 | 数值 | 同级排序 |
| 创建时间 | 日期 | 自动生成 |
| 数据总量 | 数值 | 实时计算 |

### 2.4 字段动态管理

#### 2.4.1 功能描述

每个知识表可动态增删字段，字段类型支持：**文本、数值、日期、图片、子表**。

#### 2.4.2 字段类型

| 类型 | 存储方式 | 说明 |
|------|----------|------|
| 文本（TEXT） | JSONB 字符串 | 短文本/长文本 |
| 数值（NUMBER） | JSONB 数值 | 整数或小数 |
| 日期（DATE） | JSONB 日期字符串 | 日期选择器 |
| 图片（IMAGE） | MinIO URL | 上传到 MinIO，存储访问路径 |
| 子表（SUB_TABLE） | JSONB 数组 | 内嵌子表数据，JSON 数组形式 |

#### 2.4.3 字段属性

| 属性名 | 说明 |
|--------|------|
| 字段名称 | 同一知识表内唯一 |
| 字段类型 | TEXT / NUMBER / DATE / IMAGE / SUB_TABLE |
| 是否必填 | 是 / 否 |
| 排序号 | 字段在表单/列表中的展示顺序 |
| 默认值 | 可选，新增数据时的默认值 |
| 字段配置 | JSON，预留扩展（如数值的精度、子表的字段定义等） |

#### 2.4.4 子表字段嵌套

当字段类型为**子表**时，需配置子表的字段列表（子表字段同样支持文本、数值、日期、图片类型，不无限嵌套，限制一级子表）。

#### 2.4.5 操作

- 新增字段：在已有字段列表末尾或指定位置添加
- 编辑字段：修改字段名称、类型、必填等属性（类型变更需提示数据兼容性风险）
- 删除字段：删除字段及其下所有数据值，需确认
- 字段排序：拖拽调整字段顺序

### 2.5 知识数据管理

#### 2.5.1 数据存储方案

采用 **PostgreSQL JSONB** 存储动态数据，每条记录存储为一行，核心字段用 JSONB 存放动态字段值。图片上传至 MinIO，存储 URL 于 JSONB 中。

**优势**：
- JSONB 支持索引和高效查询
- 无需 EAV 模型的复杂 JOIN
- PostgreSQL JSONB 支持 GIN 索引，满足模糊搜索需求

#### 2.5.2 数据 CRUD

| 功能点 | 说明 |
|--------|------|
| 新增数据 | 根据知识表字段定义动态渲染表单，校验必填和类型后保存 |
| 编辑数据 | 回填已有数据到动态表单，修改后保存 |
| 删除数据 | 支持单条删除和批量删除，删除关联图片（MinIO） |
| 查看详情 | 按字段定义格式化展示所有字段值 |

#### 2.5.3 分页查询

| 功能点 | 说明 |
|--------|------|
| 分页列表 | 后端分页，支持配置每页条数（默认 20） |
| 排序 | 支持按创建时间、更新时间，以及可配置的排序字段 |
| 字段列展示 | 列表展示字段可由用户自行选择显示/隐藏 |

#### 2.5.4 全局模糊搜索

| 功能点 | 说明 |
|--------|------|
| 搜索范围 | **所有知识表**的所有文本类型字段 |
| 搜索方式 | 输入关键字，后端使用 PostgreSQL `jsonb` 全文检索或 `LIKE` 模糊匹配 |
| 搜索结果 | 展示匹配的数据记录，标注所属知识表和分类 |
| 高亮展示 | 匹配关键字高亮显示 |

**搜索实现方案**：

```sql
-- 利用 PostgreSQL JSONB 进行全局搜索
SELECT * FROM knowledge_data
WHERE jsonb_data::text ILIKE '%关键字%'
  AND is_deleted = false
ORDER BY updated_at DESC
LIMIT 20 OFFSET 0;
```

> 性能优化建议：数据量较大时，对 `jsonb_data` 建立 GIN 索引，或使用 `tsvector` 全文索引。

### 2.6 地图联动

#### 2.6.1 功能描述

支持将某个知识表**配置为可联动地图展示**，并指定表中用于地图定位的关键字段（经度字段、纬度字段、地点名称字段）。

#### 2.6.2 配置项

| 配置项 | 说明 |
|--------|------|
| 是否启用地图 | 开关，默认关闭 |
| 经度字段 | 选择知识表中的数值字段 |
| 纬度字段 | 选择知识表中的数值字段 |
| 标注名称字段 | 选择知识表中的文本字段，用于地图 Marker 名称 |
| 标注描述字段 | 可选，选择文本字段用于 InfoWindow 内容 |

#### 2.6.3 地图展示

- 使用**开源地图库**（如 Leaflet + 开放街道地图 / 天地图）
- 按经纬度打点展示
- 点击标注弹出信息窗口
- 支持聚合展示（点位密集时自动聚合）
- 分页数据的点位同步展示（缩放到当前视野范围内的数据）

### 2.7 知识图谱

#### 2.7.1 功能描述

支持**按分类层级**，指定一个或多个知识表的字段作为知识图谱的**节点**，手动建立节点之间的**关系**，并以力导向图等形式**可视化展示**。

#### 2.7.2 节点配置

| 功能点 | 说明 |
|--------|------|
| 选择数据源 | 选择一个或多个知识表 |
| 指定节点名称字段 | 选择文本字段，作为图谱节点显示名称 |
| 指定节点标签字段 | 可选，用于节点的分类标签（不同颜色区分） |
| 节点生成 | 系统根据配置自动从知识表数据中提取节点 |
| 节点同步 | 知识表数据增删改时，图谱节点同步更新 |

#### 2.7.3 关系管理

| 功能点 | 说明 |
|--------|------|
| 手动建立关系 | 选择两个已有节点，建立连线 |
| 关系类型 | 支持定义关系名称（如"关联"、"属于"、"依赖"等） |
| 关系方向 | 支持单向 / 双向 |
| 编辑关系 | 修改关系名称或方向 |
| 删除关系 | 删除连线关系 |
| 批量导入关系 | 可选：通过 Excel/CSV 批量导入关系 |

#### 2.7.4 图谱展示

| 功能点 | 说明 |
|--------|------|
| 可视化渲染 | 使用 AntV G6 或 ECharts，以**力导向图**展示 |
| 节点区分 | 按知识表来源使用不同颜色区分节点 |
| 交互操作 | 支持拖拽节点、缩放画布、点击查看节点详情 |
| 筛选 | 按知识表来源筛选显示/隐藏节点 |
| 搜索定位 | 在图谱中搜索节点名称并高亮定位 |
| 导出 | 支持导出图谱为图片（PNG/SVG） |

## 3. 数据库设计

### 3.1 ER 关系概览

```
knowledge_category (分类表)
    │ 1:N
    ▼
knowledge_table (知识表定义)
    │ 1:N
    ├──────────────► knowledge_field (字段定义)
    │
```

> `jsonb_data` 示例：
> ```json

>   "name": "某地标建筑",
>   "longitude": 120.155,
>   "latitude": 30.274,
>   "build_date": "2020-01-15",
>   "photo": "http://minio-server/bucket/xxx.jpg",
>   "sub_items": [
>       { "sub_name": "子项A", "sub_count": 10 },
>       { "sub_name": "子项B", "sub_count": 20 }

**索引设计**：
```sql
-- 知识表ID索引
CREATE INDEX idx_kd_table_id ON knowledge_data(table_id);
-- JSONB GIN 索引（用于全局搜索）
CREATE INDEX idx_kd_jsonb ON knowledge_data USING GIN (jsonb_data);
-- 删除标记索引
CREATE INDEX idx_kd_deleted ON knowledge_data(is_deleted);
```

## 4. API 接口设计

### 4.1 接口规范

- Base URL: `/api/v1/knowledge`
- 请求格式: `application/json`
- 响应格式:

```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1716700000000
}
```

- 分页响应:

```json
{
  "code": 200,
  "data": {
    "content": [],
    "totalElements": 100,
    "totalPages": 5,
    "currentPage": 1,
    "pageSize": 20
  }
}
```

### 4.2 知识分类接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/categories/tree` | 获取分类树 |
| GET | `/categories/{id}` | 获取分类详情 |
| POST | `/categories` | 新增分类 |
| PUT | `/categories/{id}` | 编辑分类 |
| DELETE | `/categories/{id}` | 删除分类（校验子节点和数据） |
| PUT | `/categories/{id}/sort` | 更新排序 |

### 4.3 知识表接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/categories/{categoryId}/tables` | 按分类获取知识表列表（含数据总量） |
| GET | `/tables/{id}` | 获取知识表详情（含字段列表） |
| POST | `/tables` | 新增知识表 |
| PUT | `/tables/{id}` | 编辑知识表 |
| DELETE | `/tables/{id}` | 删除知识表及数据 |
| PUT | `/tables/{id}/map-config` | 配置地图联动 |

### 4.4 字段管理接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/tables/{tableId}/fields` | 获取字段列表 |
| POST | `/tables/{tableId}/fields` | 新增字段 |
| PUT | `/fields/{id}` | 编辑字段 |
| DELETE | `/fields/{id}` | 删除字段 |
| PUT | `/fields/sort` | 批量更新字段排序 |

### 4.5 知识数据接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/tables/{tableId}/data` | 分页查询数据 |
| GET | `/data/{id}` | 获取数据详情 |

## 7. 非功能性需求

### 7.1 性能要求

| 指标 | 要求 |
|------|------|
| 数据列表查询响应 | < 500ms（单表 < 10万条） |
| 全局搜索响应 | < 2s（总数据量 < 50万条） |
| 图谱渲染 | 节点 < 5000 时 < 3s |
| 图片上传 | 单张 < 5MB，上传 < 3s |

### 7.2 安全要求

- SQL 注入防护：使用 JPA 参数化查询，JSONB 操作注意防注入
- 文件上传校验：限制图片格式（jpg/png/gif/svg），校验文件头魔数
- XSS 防护：数据展示时进行输出编码
- 输入校验：所有接口参数做校验（`@Valid`）

### 7.3 数据一致性

- 删除知识表时级联删除数据记录、字段定义、图谱节点
- 删除字段时清理 JSONB 中对应 key 的值
- 图谱节点与数据记录保持同步（数据修改/删除时更新节点）

## 8. 扩展性考虑（不增加中间件）

以下扩展基于现有技术栈，不引入新的中间件：

### 8.1 字段类型扩展

通过 `field_type` 枚举扩展新类型（如 FILE、VIDEO、RICH_TEXT），只需：
1. 枚举中新增类型
2. 前端添加对应的渲染组件和输入组件
3. 存储仍使用 JSONB，无需改表结构

### 8.2 数据导入导出

- 知识表数据支持导出为 Excel/CSV
- 支持按模板批量导入（Apache POI）

### 8.3 数据版本/审计

- 在 `knowledge_data` 表中增加 `version` 字段
- 新增 `knowledge_data_history` 表记录变更历史

### 8.4 图谱自动关系发现

- 基于字段值相似度（如相同的地点名称）自动推荐关系
- 仍使用 PostgreSQL 的 `pg_trgm` 扩展（PostgreSQL 自带，非额外中间件）

## 10. 风险与注意事项

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| JSONB 数据量增长导致查询变慢 | 搜索响应超时 | 建立 GIN 索引，考虑按时间分表归档 |
| 动态字段类型变更导致数据不兼容 | 数据读取异常 | 类型变更时前端做兼容处理，后端保留原始值 |
| 图谱节点过多导致前端渲染卡顿 | 用户体验差 | 节点数 > 1000 时启用虚拟化/分批渲染 |
| MinIO 单点故障 | 图片无法访问 | MinIO 集群部署或定期备份 |
